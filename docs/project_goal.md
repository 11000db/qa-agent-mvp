# Project Goal

## 프로젝트 목적
이 프로젝트의 목적은 QA 실무 경험을 AI 기반 워크플로우로 구조화하는 것이다.

요구사항 또는 티켓을 입력하면,
QA 관점으로 테스트 포인트를 분석하고,
리스크 기반 테스트 케이스와 Playwright 테스트 초안을 생성하는
QA Agent MVP를 만드는 것을 목표로 한다.

## 왜 이 프로젝트를 만드는가
실무에서 요구사항 분석과 테스트 케이스 작성은 시간이 많이 들고,
사람마다 테스트 품질 편차가 발생한다.

특히 아래와 같은 문제가 자주 발생한다.

- 요구사항을 읽고 테스트 포인트를 정리하는 데 시간이 오래 걸린다.
- 테스트 케이스 품질이 작성자에 따라 달라진다.
- happy path 위주로 작성되고 negative / edge case가 빠지기 쉽다.
- QA 노하우가 개인에게 축적되고 팀 자산으로 남지 않는다.
- 자동화 테스트로 연결되는 과정이 단절되어 있다.

이 프로젝트는 이런 문제를 줄이고,
QA 노하우를 구조화해서 재사용 가능한 형태로 만드는 것을 목표로 한다.

## 핵심 목표
- 요구사항 분석 자동화 보조
- 리스크 기반 테스트 케이스 생성
- QA 판단 기준이 반영된 테스트 설계 흐름 정리
- Playwright 테스트 초안 생성 연결
- 실행 결과를 확인할 수 있는 최소 데모 완성

## 포트폴리오 목표
이 프로젝트는 단순한 코드 생성기가 아니라,
QA 엔지니어가 AI와 자동화를 활용해 품질 설계 역량을 확장한 사례로 정리하는 것을 목표로 한다.

최종적으로는 아래 흐름을 하나의 데모로 보여줄 수 있어야 한다.

요구사항 입력
→ 테스트 포인트 분석
→ 테스트 케이스<span class="cursor">█</span>
